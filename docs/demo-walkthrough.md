# Demo Walkthrough

## 1. Demo Objective

Show judges that the product is not just a safety alarm app, but a complete tourist travel-assistance platform with proactive prevention, emergency support, and admin operations visibility.

## 2. Suggested Demo Roles

- **Presenter 1:** explains tourist journey and business value.
- **Presenter 2:** operates the mobile app.
- **Presenter 3:** operates the admin dashboard and explains architecture/security.

## 3. Judge-Friendly Demo Story

### Step 1: Plan a trip
1. Open the tourist app home screen.
2. Select a destination city.
3. Add attractions, restaurants, and hotel options to the itinerary.
4. Show safety score, local fare guidance, and emergency support overview.

### Step 2: Start the trip
1. Start the itinerary for the day.
2. Show live route navigation.
3. Explain that the system is periodically evaluating the user location against unsafe zones and event alerts.

### Step 3: Trigger a safety advisory
1. From the admin dashboard, create an unsafe zone for a protest or theft hotspot.
2. Return to the mobile app and show a geo-fence warning.
3. Demonstrate a safer alternative route and a contextual advisory like “avoid after 8 PM.”

### Step 4: Show resilience features
1. Open offline emergency information.
2. Show cached contacts for police, hospital, embassy, and help center.
3. Mention that queued updates sync when connectivity returns.

### Step 5: Trigger an emergency
1. Press SOS in the tourist app.
2. Show countdown/cancel state to reduce accidental triggers.
3. Show that the app sends location, trip context, and emergency message.

### Step 6: Admin response
1. Switch to the admin dashboard.
2. Show live SOS event on the map.
3. Assign a simulated response team.
4. Send a direct advisory back to the tourist.
5. Mark the case acknowledged.

### Step 7: Incident record and analytics
1. Convert the SOS to an incident case or show a reported theft case.
2. Display analytics: incident count by zone, active alerts, average response time.
3. Show heatmap or hotspot visualizations.

### Step 8: Trust and privacy close
1. Show emergency contacts and consent settings.
2. Open the document vault with passport/ticket placeholders.
3. Explain encryption, consent-based tracking, and admin audit logging.

## 6. Presentation Tips

- Simulate external data instead of relying on unstable live APIs.
- Use seeded unsafe zones, fake incidents, and sample places for a smooth demo.
- Highlight that the MVP is realistic because each feature can be mapped to standard app capabilities and public APIs.
- Emphasize both prevention and response; that makes the concept stronger than a normal travel app.
